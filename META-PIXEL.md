# מדידת Meta Pixel באתר קרנף נדל״ן

מסמך זה מסביר, בשפה פשוטה, מה האתר מודד ושולח למערכת הפרסום של מטא (פייסבוק/אינסטגרם) — גם למי שלא מכיר את עולם הפיקסלים.

**מזהה הפיקסל:** `1659334891302781`
**איפה רואים את הנתונים:** [Meta Events Manager](https://business.facebook.com/events_manager2) → בחירת הפיקסל → לשונית "Test Events" (לבדיקה בזמן אמת) או "Overview" (סיכום).
**כלי בדיקה מהיר:** תוסף הכרום [Meta Pixel Helper](https://chromewebstore.google.com/detail/meta-pixel-helper/fdgfkebogiimcoedlicjlajpkdmockpc) — מראה בזמן אמת אילו אירועים נשלחים בכל עמוד.

---

## מה הפיקסל עושה?

בכל פעם שמישהו נכנס לאתר, עובר בין עמודים, לוחץ על כפתור או משאיר פרטים — האתר שולח "אירוע" למטא. כך אפשר:
- לדעת אילו עמודים ופעולות הכי פופולריים.
- למדוד כמה לידים הגיעו מכל מודעה.
- לבנות קהלים לפרסום (למשל: "כל מי שנכנס לעמוד ליווי משקיעים ולא השאיר פרטים").

**חשוב:** כל אירוע נשלח עם **שם בעברית** (`content_name`) ו**קטגוריה בעברית** (`content_category`), כדי שהדוחות יהיו קריאים וברורים.

---

## האירועים שהאתר שולח

### 1. צפייה בעמוד — `PageView`
נשלח **בכל כניסה לעמוד ובכל מעבר בין עמודים** (גם ניווט פנימי בתפריט).
מגיע עם שם העמוד בעברית, למשל:
| עמוד | שם שנשלח |
|---|---|
| דף הבית | דף הבית |
| /course | הדרך לדירה — הקורס |
| /premium | ליווי משקיעים פרימיום |
| /program | הדרך לדירה — עמוד תוכנית |
| /about | אודות — סיפורו של קרנף |
| /testimonials | סיפורי הצלחה |
| /contact | צור קשר |
| /blog | בלוג — ידע ותובנות |

### 2. צפייה בעמוד שירות — `ViewContent`
נשלח **בנוסף** ל-PageView כשנכנסים לעמוד **ליווי משקיעים** או **הקורס**.
משמש כדי שמודעות שמובילות לעמודים האלה יוכלו לבצע אופטימיזציה ("להביא עוד אנשים שצופים בעמוד").

### 3. ליד — `Lead`  ⭐ (האירוע הכי חשוב)
נשלח **כשמישהו משאיר פרטים בטופס בהצלחה**. מגיע עם שם הטופס בעברית:
| מקור הטופס | שם שנשלח | קטגוריה |
|---|---|---|
| טופס עמוד ליווי משקיעים | טופס ליווי משקיעים פרימיום | ליווי משקיעים |
| טופס יצירת קשר / תחתית העמוד | טופס יצירת קשר באתר | יצירת קשר |
| טופס וובינר | הרשמה לוובינר | וובינר |

> זה האירוע שכדאי להגדיר במטא כ"המרה" (Conversion) ולבנות עליו את המודעות.

### 4. יצירת קשר — `Contact`
נשלח בלחיצה על:
- **וואטסאפ** (כל כפתור/קישור וואטסאפ באתר) → נשלח "וואטסאפ"
- **טלפון** → נשלח "טלפון"
- **אימייל** → נשלח "אימייל"

### 5. לחיצה על כפתור — `לחיצה_על_כפתור` (אירוע מותאם)
נשלח בלחיצה על כפתורים וקישורים חיצוניים חשובים. מגיע עם **הטקסט של הכפתור** (`content_name`) והעמוד שממנו נלחץ.

### 6. רשת חברתית — `רשת_חברתית` (אירוע מותאם)
נשלח בלחיצה על אייקון של אינסטגרם / פייסבוק / יוטיוב / טיקטוק / ספוטיפיי.

### 7. עומק גלילה — `עומק_גלילה` (אירוע מותאם)
נשלח כשגולשים 25% / 50% / 75% / 100% מהעמוד — מודד עד כמה אנשים באמת קוראים את התוכן.

## איך זה בנוי בקוד (לטכני)
- `src/lib/pixel.ts` — כל פונקציות השליחה + התוויות בעברית.
- `src/components/PixelTracker.tsx` — עוקב אחרי מעברי עמודים, קליקים וגלילה. מורכב פעם אחת ומכסה את כל האתר.
- `src/lib/leadSubmission.ts` — יורה את אירוע ה-`Lead` בכל שליחת טופס מוצלחת.
- `index.html` — טעינת הפיקסל (`fbq('init', ...)`). ה-PageView עצמו נשלח מה-PixelTracker כדי לתפוס גם ניווט פנימי.

**לשנות מזהה פיקסל?** יש לעדכן ב-`index.html` (שורת `fbq('init', ...)` + הקישור ב-`noscript`).
